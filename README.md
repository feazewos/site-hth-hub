# site-hth-hub

## 项目简介

这是一个用于归档和发布多个独立 HTML 页面的仓库。本仓库不针对任何特定网站或域名，仅作为一组静态 HTML 文件的集中存放与展示平台。所有页面均为独立编写，适用于学习、参考、原型展示或轻量级文档发布等场景。

## 目录说明

```
site-hth-hub/
├── pages/          # 存放各个独立的 HTML 页面文件
├── assets/         # 页面引用的静态资源（CSS、JS、图片等）
├── docs/           # 额外说明文档或参考材料（可选）
└── README.md       # 本文件
```

- `pages/` 目录下每个 HTML 文件即为一个独立的页面，文件名应反映页面内容或用途。
- `assets/` 目录按需组织，建议将共用样式、脚本或图片存放于此，便于维护。
- `docs/` 目录为非必需，可存放与页面相关的笔记、设计稿说明或使用指引。

## 页面归档说明

本仓库中的 HTML 页面均为独立归档，彼此之间没有依赖关系。每个页面都包含完整的 HTML 结构，可以直接在浏览器中打开运行。部分页面可能引用 `assets/` 目录下的资源，请确保文件路径正确。

页面归档的主要目的是保存和展示已完成或进行中的 HTML 作品，方便随时回溯、分享或部署。归档内容不涉及任何商业用途，也不包含任何具体网站的镜像或复制。

## 维护说明

- 新增页面：将 HTML 文件放入 `pages/` 目录，并在必要时更新 `assets/` 中的资源。建议文件名使用英文小写与连字符，如 `example-page.html`。
- 删除页面：直接移除对应文件，并清理不再使用的资源。
- 资源引用：尽量保持相对路径引用，避免硬编码绝对地址。
- 版本控制：本仓库使用 Git 管理，每次更新请附带清晰的提交信息，说明新增或修改的内容。
